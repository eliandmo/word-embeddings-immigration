PREPROCESSING

Preprocessing Pipeline

Academic abstracts exported from Scopus contain noise that distorts NLP computations. A two-version strategy is used: raw_abstracts for display; clean (preprocessed) for all computations.

  • 1Remove structural headers — Objectives, Study design, Methods, Results, Conclusions are section labels embedded in structured abstracts (P1). They carry no thematic content.
  • 2Lowercase — avoids treating Immigration and immigration as different tokens.
  • 3Remove diacritics — Argüellová → arguellova; prevents OOV caused by encoding differences.
  • 4Remove numbers and special chars — 1996, 287(g), © carry no semantic embedding value.
  • 5Remove stopwords + tokens < 3 chars — prevents generic function words from dominating mean embedding vectors.
  • 6Strip extra whitespace — cosmetic normalization.
In [7] — Preprocessing pipeline
# ── Preprocessing pipeline ────────────────────────────────────
from sklearn.feature_extraction.text import ENGLISH_STOP_WORDS

STOPWORDS = set(ENGLISH_STOP_WORDS) | {
    # Abstract structural headers
    "objectives", "methods", "results", "conclusions",
    "background", "design", "findings", "context",
    # Academic filler verbs
    "find", "found", "show", "argue", "examine",
    "consider", "include", "use", "uses", "used",
    # Common filler
    "also", "however", "thus", "may", "can",
    "will", "would", "could", "three", "one", "two",
}

_HEADERS = re.compile(
    r"\b(Objectives?|Study\s+design|Methods?|Results?|Conclusions?|"
    r"Background|Introduction|Discussion|Abstract|Findings)\b",
    re.IGNORECASE
)

def preprocess(text):
    text = _HEADERS.sub(" ", text)                      # step 1
    text = text.lower()                                   # step 2
    text = unicodedata.normalize("NFKD", text)\
           .encode("ascii", "ignore").decode("utf-8")  # step 3
    text = re.sub(r"[^a-z\s]", " ", text)              # step 4
    tokens = [t for t in text.split()
              if t not in STOPWORDS and len(t) >= 3]  # step 5
    return " ".join(tokens)                               # step 6

clean = {k: preprocess(v) for k, v in raw_abstracts.items()}
clean_list = [clean[k] for k in doc_keys]

# Before / After report
for k in doc_keys:
    n_r = len(raw_abstracts[k].split())
    n_c = len(clean[k].split())
    print(f"{k}: raw={n_r}  clean={n_c}  reduction={100*(n_r-n_c)/n_r:.1f}%")

§ 1

From Frequency-Based to Embedding-Based Representations

Bag-of-Words and TF-IDF encode text as sparse vectors of token counts or reweighted frequencies. Their similarity signal depends entirely on shared vocabulary. Two abstracts discussing the same immigration phenomenon using different disciplinary vocabularies — one writing deportation, another removal — score zero overlap under TF-IDF because they share no surface tokens.

Word embeddings encode each word as a dense vector learned from co-occurrence patterns across large corpora. Words appearing in similar linguistic environments are mapped to nearby points in vector space regardless of surface form, capturing semantic proximity rather than lexical overlap.

For this corpus, researchers from public health (P1), political science (P6), and security studies (P5) describe overlapping enforcement realities with entirely different technical vocabularies. Embedding-based similarity recovers those relationships; TF-IDF misses them.

BoW / TF-IDF
Sparse vectors · Fixed vocabulary · Lexical overlap only
Fast, interpretable, language-specific
GloVe Embeddings
Dense vectors · Semantic geometry · Context-aware
Captures synonyms, paraphrases, cross-discipline vocabulary

Analogy Reasoning

Analogy queries use vector arithmetic to probe relational structure: vb − va + vc ≈ vd. Helper functions from the course document are used.

In [17] — Helper functions + Analogy 1
def require(m, tokens):
    return [t for t in tokens if t not in m.key_to_index]

def run_analogy(m, positive, negative, topn=5):
    miss = require(m, positive + negative)
    if miss:
        print(f"  ⚠ Missing tokens: {miss}")
        return
    results = m.most_similar(positive=positive,
                              negative=negative, topn=topn)
    df = pd.DataFrame(results, columns=["Result", "Cosine sim"])
    df["Rank"] = range(1, len(df)+1)
    df["Cosine sim"] = df["Cosine sim"].round(4)
    print(df[["Rank", "Result", "Cosine sim"]].to_string(index=False))

# ── Mean embedding similarity ────────────────────────────────────
def mean_vec(m, text):
    toks = [t for t in text.split() if t in m.key_to_index]
    return np.mean([m[t] for t in toks], axis=0) if toks else None

def cos(u, v):
    nu, nv = np.linalg.norm(u), np.linalg.norm(v)
    return float(np.dot(u,v)/(nu*nv)) if nu>0 and nv>0 else 0.

vecs   = {k: mean_vec(glove, clean[k]) for k in doc_keys}
M_emb  = np.zeros((n_docs, n_docs))
for i, ki in enumerate(doc_keys):
    for j, kj in enumerate(doc_keys):
        M_emb[i, j] = cos(vecs[ki], vecs[kj])

§ 8

Conceptual Reflection

What embeddings capture that TF-IDF does not

TF-IDF scores documents as vectors over a fixed vocabulary. Two abstracts discussing identical policy phenomena using different disciplinary vocabularies score near-zero similarity. In this corpus, P1 (public health, prenatal care at the border) and P4 (Catholic institutions serving immigrants) both describe consequences of the same immigration enforcement crisis — institutional barriers to services, vulnerability of immigrant communities, erosion of access — yet their TF-IDF similarity is 0.0644. Mean-embedding cosine reaches 0.9267 for the same pair, because healthcare, barriers, immigrant co-occur with similar contextual words to institutions, enforcement, services across the Wikipedia/Gigaword corpus. The embedding captures the shared policy context even when disciplinary lexicons diverge entirely.

One key limitation of GloVe / Word2Vec

Both models assign a single static vector to each word regardless of context. The word border in P5 refers to a physical enforcement zone in the Sonoran desert with smuggling networks and detention facilities; in P6 it refers to an abstract policy object in a public opinion study about American values. Both usages receive the same 100-dimensional vector. This context-blindness is the central limitation that motivates contextual embeddings (BERT, RoBERTa, GPT), where each word's representation is dynamically constructed based on its sentence-level context. For this narrow-domain corpus, the single-sense problem is relatively contained; in a cross-domain corpus it would introduce substantially more noise.

When embeddings are preferable

Embeddings are preferable when: (1) the corpus spans multiple disciplines describing the same phenomena with different vocabularies — exactly the case here; (2) the goal is semantic search, clustering, or paraphrase detection rather than exact keyword retrieval; (3) the corpus is multilingual — multilingual models handle cross-language similarity natively; (4) documents are short — TF-IDF statistics become unstable for short texts while embeddings retain semantic signal. TF-IDF remains preferable when interpretability is paramount (the top tokens per document are directly meaningful, as shown in Section 7), when vocabulary overlap is genuinely informative, or when computational resources are severely constrained.
